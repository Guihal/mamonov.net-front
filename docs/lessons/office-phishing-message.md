# Урок: Фишинг в мессенджере

**Slug**: `office-phishing-message`
**Категория**: Офис (office)
**Порядок**: 1 из 3
**Угроза**: Фишинговые ссылки в мессенджерах

---

## Идея

Пользователь на «работе» — открыт мессенджер с рабочим чатом. Приходит сообщение от «Димы коллеги» с укороченной ссылкой и текстом «Срочно проверь отчёт по проекту!». Ссылка ведёт на фейковый сайт банка. Нужно не кликать, а проверить URL и сообщить о подозрительном сообщении.

---

## Программы

`messenger`, `browser`

---

## Сценарий по шагам

### Шаг 0 — старт

- Открыт мессенджер, чат «Рабочий чатик» (групповой)
- Видны несколько сообщений с утра (нейтральные: «Всем привет», «Созвон в 15:00», «Кто берёт кофе?»)
- Приходит новое сообщение от **«Дима коллега»**:
  > «Привет всем! Срочно нужна помощь — проверьте отчёт по Q1, там что-то не сходится. Вот файл: https://bit.ly/3xAbc99 — откройте быстрее, дедлайн через час!»
- LessonAlert: «Ты на работе, общаешься в корпоративном мессенджере. Обращай внимание на подозрительные ссылки.»

**Маскот — очередь реплик (введение в сцену):**

1. `{ text: 'Обычный рабочий день. Ты в офисе, открыт корпоративный мессенджер — коллеги обсуждают текущие задачи.', emotion: 'neutral', delay: 1500 }` — авто
2. `{ text: 'Мессенджеры на работе — привычный инструмент. Файлы, ссылки, обсуждения — всё здесь.', emotion: 'neutral' }` — ждёт клика
3. `{ text: 'О, новое сообщение от Димы! «Срочно проверь отчёт, дедлайн через час!» И ссылка...', emotion: 'neutral' }` — ждёт клика
4. `{ text: 'Подожди. Срочность, давление, «проверьте быстрее» — это типичные приёмы, чтобы ты не думал и просто кликнул. Не торопись!', emotion: 'worried' }` — ждёт клика

---

### Шаг 1 — пользователь анализирует ссылку

Пользователь работает в OS самостоятельно:

- Открывает браузер и переходит на `tools://url-checker`, вводит `https://bit.ly/3xAbc99` → `onUrlNavigate('tools://url-checker')` → шаг 2
- Открывает личный чат «Дима личка» в мессенджере и спрашивает → `onMessageSend('dima-personal')` → шаг 3 (быстрый путь)
- Просто не кликал ссылку в мессенджере — маскот через таймер попросит что делать

QuickActions на шаге 1 **отсутствуют** — всё делается в мессенджере и браузере.

**Маскот:**

1. `{ text: 'Укороченная ссылка — bit.ly скрывает реальный адрес. Никогда не переходи по таким ссылкам вслепую — сначала проверь, куда она ведёт.', emotion: 'neutral' }` — ждёт клика

---

### Шаг 2 — проверка URL

`tools://url-checker` — пользователь вводит `https://bit.ly/3xAbc99`.

Результат (мок): **Опасно** — ссылка перенаправляет на `fake-yourbank-login.ru`, домен зарегистрирован 2 дня назад, содержит форму для кражи учётных данных.

**Маскот:**

1. `{ text: 'Вот оно! bit.ly/3xAbc99 → fake-yourbank-login.ru. Это явно не рабочий отчёт по Q1.', emotion: 'neutral' }` — ждёт клика
2. `{ text: 'Кстати, даже если бы ссылка выглядела нормально — рабочие документы в компании не хранят на случайных сайтах. Их отправляют через корпоративное облако.', emotion: 'neutral' }` — ждёт клика

Quick Action: **«Позвонить в ИБ»** → шаг 3 (нельзя в OS)

---

### Шаг 3 — правильная реакция

После проверки ссылки / уточнения у Димы лично → маскот хвалит.

**Маскот:**

1. `{ text: 'Отлично! Ты не перешёл по ссылке под давлением дедлайна.', emotion: 'happy' }` — ждёт клика
2. `{ text: 'К слову — аккаунт Димы скорее всего взломали. Именно так и работают атаки через доверенных людей.', emotion: 'neutral' }` — ждёт клика

Quick Action: **«Завершить урок»** → `complete()` (официальное подтверждение завершения, нельзя в OS)

---

### Завершение — complete()

**Маскот — финальные реплики:**

1. `{ text: 'Урок пройден! Ты устоял перед давлением «срочно, дедлайн!».', emotion: 'happy' }` — ждёт клика
2. `{ text: 'Правило 1: укороченные ссылки всегда проверяй перед переходом — они скрывают реальный адрес.', emotion: 'happy' }` — ждёт клика
3. `{ text: 'Правило 2: срочность и давление — главные инструменты фишинга. Чем сильнее торопят, тем подозрительнее.', emotion: 'happy' }` — ждёт клика
4. `{ text: 'Правило 3: если ссылку прислал «знакомый» — уточни у него другим способом (позвони, напиши лично).', emotion: 'happy', delay: 6000 }` — авто

## Ошибки (fail-сценарии)

### ❌ Кликнуть по ссылке в мессенджере

Ивент: `onLinkClick(chatId, url)` → если `url.includes('bit.ly')` → `fail(...)`

**Маскот:**

1. `{ text: 'Ой!', emotion: 'sad', delay: 600 }` — авто
2. `{ text: 'Ты перешёл по ссылке прямо из мессенджера. Укороченная ссылка скрывала фишинговый сайт — теперь он знает твой IP и может определить браузер.', emotion: 'sad' }` — ждёт клика

---

### ❌ Открыть ссылку в браузере вручную

Ивент: `onUrlNavigate(url)` → если `url.includes('bit.ly') || url.includes('fake-yourbank')` → `fail(...)`

**Маскот:**

1. `{ text: 'Стоп...', emotion: 'worried', delay: 700 }` — авто
2. `{ text: 'Ты сам вписал фишинговый адрес в браузер. Любопытство — понятно, но это ровно то, на что рассчитывает атака.', emotion: 'sad' }` — ждёт клика

---

### ❌ Ввести данные на фейковом сайте

Ивент: `onFormSubmit(url, data)` → `fail(...)`

**Маскот:**

1. `{ text: 'Нет!', emotion: 'sad', delay: 500 }` — авто
2. `{ text: 'Ты ввёл свои данные на фишинговом сайте. Теперь они у злоумышленника. Именно так воруют корпоративные аккаунты и деньги со счетов.', emotion: 'sad' }` — ждёт клика

## Quick Actions (полный список)

Только действия, которые нельзя выполнить в симулированном компьютере:

| id                | Иконка                  | Подпись        | Когда | Действие     |
| ----------------- | ----------------------- | -------------- | ----- | ------------ |
| `call-security`   | `i-lucide-phone`        | Позвонить в ИБ | шаг 2 | → шаг 3      |
| `complete-lesson` | `i-lucide-check-circle` | Завершить урок | шаг 3 | `complete()` |

Всё остальное пользователь делает сам в OS:

- Открывает `tools://url-checker` в браузере и проверяет ссылку bit.ly
- Открывает личный чат с Димой в мессенджере и уточняет
- Не кликает ссылку в мессенджере (видно в UI)

## Данные для LessonConfig

```ts
const phishingMessageConfig: LessonConfig = {
  id: 'office-phishing-message',
  categoryId: 'office',
  programs: ['messenger', 'browser'],

  messengerChats: [
    {
      id: 'work-chat',
      name: 'Рабочий чатик',
      isGroupChat: true,
      messages: [
        { id: '1', sender: 'Катя HR', text: 'Всем привет!', isOwn: false, timestamp: '08:32' },
        {
          id: '2',
          sender: 'Паша разраб',
          text: 'Созвон в 15:00, не забудьте',
          isOwn: false,
          timestamp: '09:01'
        },
        { id: '3', sender: 'Ты', text: 'Буду', isOwn: true, timestamp: '09:03' },
        {
          id: '4',
          sender: 'Дима коллега',
          text: 'Привет всем! Срочно нужна помощь — проверьте отчёт по Q1, там что-то не сходится. Вот файл: https://bit.ly/3xAbc99 — откройте быстрее, дедлайн через час!',
          isOwn: false,
          timestamp: '10:17'
        }
      ]
    },
    {
      id: 'dima-personal',
      name: 'Дима коллега',
      messages: [],
      isGroupChat: false
    }
  ],

  events: {
    messenger: {
      onLinkClick: (chatId, url) => {
        mascot.enqueue([
          { text: 'Ой!', emotion: 'sad', delay: 600 },
          {
            text: 'Ты перешёл по ссылке прямо из мессенджера. Укороченная ссылка скрывала фишинговый сайт.',
            emotion: 'sad'
          }
        ])
        fail('Вы перешли по фишинговой ссылке в мессенджере!')
      },
      onMessageSend: (chatId, text) => {
        if (chatId === 'dima-personal' && text.trim().length > 0) {
          // Дима отвечает: "Я ничего не отправлял!"
          simulateDimaReply()
          nextStep(3)
        }
      }
    },
    browser: {
      onUrlNavigate: (url) => {
        if (url.includes('bit.ly') || url.includes('fake-yourbank')) {
          mascot.enqueue([
            { text: 'Стоп...', emotion: 'worried', delay: 700 },
            {
              text: 'Ты сам вписал фишинговый адрес в браузер. Любопытство — понятно, но это ровно то, на что рассчитывает атака.',
              emotion: 'sad'
            }
          ])
          fail('Вы открыли фишинговый сайт!')
        }
      },
      onFormSubmit: (url, data) => {
        mascot.enqueue([
          { text: 'Нет!', emotion: 'sad', delay: 500 },
          {
            text: 'Ты ввёл свои данные на фишинговом сайте. Именно так воруют корпоративные аккаунты.',
            emotion: 'sad'
          }
        ])
        fail('Вы ввели данные на фишинговом сайте!')
      }
    }
  },

  mascotPhrases: {
    greet: 'Ты на работе. Срочное сообщение от коллеги со ссылкой — не торопись кликать!'
  }
}
```
