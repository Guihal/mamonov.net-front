# Урок: Фейковый портал авторизации

**Slug**: `wifi-captive-portal`
**Категория**: Общественный Wi-Fi (public-wifi)
**Порядок**: 3 из 3
**Угроза**: Captive Portal фишинг

---

## Идея

Пользователь подключается к Wi-Fi в торговом центре. Браузер автоматически открывает «портал авторизации» с кнопками «Войти через ВКонтакте», «Войти через Google» и формой «Email + пароль». Настоящие captive-порталы не просят пароли от внешних сервисов — только номер телефона или согласие с правилами. Нужно это распознать.

## Сценарий по шагам

### Шаг 0 — старт

- Браузер автоматически открывает `captive://mall-wifi` (`CaptivePortalPage.vue`) — «Добро пожаловать в Wi-Fi ТЦ "МегаМолл"»
- На странице:
  - Кнопка (большая, синяя): «Войти через ВКонтакте» → открывает форму с полем «Логин ВК» + «Пароль»
  - Кнопка (большая, красная): «Войти через Google» → форма «Email + Пароль Google»
  - Разделитель «или»
  - Форма «Email + пароль» (универсальная)
  - Мелким шрифтом внизу: домен `mall-free-wifi-auth.ru`
- LessonAlert: «Подключился к Wi-Fi в торговом центре. Открылся портал для входа.»

**Маскот — очередь реплик:**
1. `{ text: 'Стоп! Перед тем как нажать что-то — посмотри на страницу внимательно. Что именно он у тебя просит?', emotion: 'worried' }` — ждёт клика

Ивент: `onFormSubmit(url, { vkLogin, vkPassword })` → если `url.includes('mall-free-wifi')` и есть данные → `fail(...)`

## Браузерные сайты для LessonConfig

```ts
browserSites: [
  {
    url: 'captive://mall-wifi',
    title: 'Wi-Fi авторизация — ТЦ МегаМолл',
    component: CaptivePortalPage
  }
]
```

`CaptivePortalPage.vue` — стилизованный фейковый портал:
- Шапка: логотип «МегаМолл Wi-Fi»
- Три варианта «входа»: VK (синяя кнопка), Google (красная кнопка), Email/пароль (форма)
- Мелкий текст внизу с доменом `mall-free-wifi-auth.ru`
- Каждая форма при submit → emit → inject browserEvents → `onFormSubmit`

**Важно**: кнопки «Войти через VK» и «Войти через Google» НЕ используют OAuth — они просто показывают обычную форму ввода. Это ключевой учебный момент: визуально похоже на OAuth, но работает как простой сбор данных.

## LessonConfig

```ts
const captivePortalConfig: LessonConfig = {
  id: 'wifi-captive-portal',
  categoryId: 'public-wifi',
  programs: ['browser'],

  browserSites: [
    { url: 'captive://mall-wifi', title: 'Wi-Fi авторизация — ТЦ МегаМолл', component: CaptivePortalPage }
  ],

  quickActions: [
    {
      id: 'check-domain',
      icon: 'i-lucide-shield-check',
      label: 'Проверить домен портала',
      onClick: () => {
        openBrowserTab('tools://url-checker')
        nextStep(2)
      }
    },
    {
      id: 'ask-staff-mall',
      icon: 'i-lucide-user-check',
      label: 'Уточнить у персонала',
      onClick: () => {
        mascot.enqueue([
          { text: 'Сотрудник ТЦ: «Наш портал только просит номер телефона. Он никогда не спрашивает пароли от ВКонтакте или Google.»', emotion: 'neutral' }
        ])
        nextStep(2)
      }
    },
    {
      id: 'read-fine-print',
      icon: 'i-lucide-file-text',
      label: 'Читать мелкий шрифт',
      onClick: () => {
        mascot.enqueue([
          { text: 'Видишь внизу? «mall-free-wifi-auth.ru». Это не сайт ТЦ «МегаМолл» — это посторонний домен, который притворяется порталом аэропорта.', emotion: 'worried' }
        ])
        nextStep(2)
      }
    },
    {
      id: 'close-portal',
      icon: 'i-lucide-x-circle',
      label: 'Закрыть портал',
      onClick: () => nextStep(3)
    },
    {
      id: 'use-mobile-data',
      icon: 'i-lucide-signal',
      label: 'Использовать мобильный интернет',
      onClick: () => complete()
    }
  ],

  events: {
    browser: {
      onFormSubmit: (url, data) => {
        if (!url.includes('mall-free-wifi')) return

        const isGoogle = !!(data.googleEmail || data.googlePassword)
        const isVk = !!(data.vkLogin || data.vkPassword)

        if (isGoogle) {
          mascot.enqueue([
            { text: 'О нет!', emotion: 'sad', delay: 600 },
            { text: 'Пароль Google — ключ ко всей экосистеме: Gmail, Drive, Photos, Play Store. Один из самых ценных паролей для злоумышленника.', emotion: 'sad' }
          ])
        } else if (isVk) {
          mascot.enqueue([
            { text: 'Нет!', emotion: 'sad', delay: 500 },
            { text: 'Ты ввёл пароль ВКонтакте на стороннем сайте. Кнопка «Войти через VK» на фейковом портале — это просто форма сбора данных, не OAuth.', emotion: 'sad' },
            { text: 'Мошенник теперь имеет доступ к твоим сообщениям и, возможно, привязанной карте.', emotion: 'sad' }
          ])
        } else {
          mascot.enqueue([
            { text: 'Ой...', emotion: 'sad', delay: 600 },
            { text: 'Через email мошенник может сбросить пароли от большинства твоих аккаунтов. Почта — это мастер-ключ.', emotion: 'sad' }
          ])
        }

        fail('Вы ввели учётные данные на фишинговом captive-портале!')
      }
    }
  },

  mascotPhrases: {
    greet: 'Портал Wi-Fi просит пароль от ВКонтакте? Зачем торговому центру твой пароль от соцсети?'
  }
}
```

## Визуал CaptivePortalPage.vue

Стиль «офциального» корпоративного сайта (чтобы выглядел убедительно):
- Чистый белый фон, синяя шапка с «лого» МегаМолл
- Кнопка VK: синяя, с иконкой «ВКонтакте» — выглядит как настоящая
- Кнопка Google: белая с тенью, иконка Google — выглядит как настоящая
- Форма Email/пароль под разделителем «или»
- Кнопка «Войти» → submit
- Мелкий серый текст внизу: «Авторизация предоставлена mall-free-wifi-auth.ru» — специально нечитаемым размером
