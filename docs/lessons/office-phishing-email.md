# Урок: Фишинговое письмо

**Slug**: `office-phishing-email`
**Категория**: Офис (office)
**Порядок**: 2 из 3
**Угроза**: Фишинговые email-письма с вложениями и поддельными ссылками

---

## Идея

В почтовом ящике лежит письмо «от начальника» Сергея Петровича с вложением `Отчёт_Q4.xlsx.exe` и ссылкой на «корпоративный портал» `corp-portall.ru` (двойная «l»). Нужно заметить поддельный домен отправителя, не открывать вложение с двойным расширением и сообщить в службу ИБ.

---

## Программы

`mail`, `browser`

---

## Сценарий по шагам

### Шаг 0 — старт

- Открыта почта, во входящих 3 письма (2 нейтральных + 1 подозрительное)
- Подозрительное письмо от **«Сергей Петрович Директор»** `<director@corp-holdings.net>` (настоящий адрес компании — `corp.ru`):
  - Тема: «СРОЧНО: Квартальный отчёт Q4 — подписать до 18:00»
  - Тело письма: «Иван, добрый день! Прошу срочно ознакомиться с отчётом и подписать. Вложение: `Отчёт_Q4.xlsx.exe`. Также сверьтесь с данными на нашем портале: http://corp-portall.ru/q4-report»
  - Отправлено в 17:43 (рабочий день почти заканчивается — давление)
- LessonAlert: «Внимательно смотри на адрес отправителя и расширение вложений.»

**Маскот — очередь реплик:**

1. `{ text: 'Письмо «от начальника». Срочно, до 18:00. Час пик для фишинга — когда люди устали и торопятся.', emotion: 'worried' }` — ждёт клика

---

### Шаг 1 — анализ письма

Пользователь работает в OS самостоятельно:

- Читает адрес отправителя в почте (виден в заголовке письма)
- Открывает браузер и переходит на `tools://url-checker` для проверки `corp-portall.ru` (живая ссылка в теле  письма)
- Открывает вложение — `tools://file-checker` для анализа `.xlsx.exe` (виден в UI почты)
- События `onUrlNavigate`, `onAttachmentOpen` — отслеживают прогресс и ошибки

QuickActions на шаге 1 **отсутствуют** — всё делается в браузере и почте.

**Маскот при открытии письма:**

1. `{ text: 'Подожди — адрес отправителя «corp-holdings.net». Твоя компания называется corp.ru. Это разные домены. Директор пишет с рабочего адреса, а не с «holdings» на непонятном домене.', emotion: 'worried' }` — ждёт клика

---

### Шаг 2 — проверка одного из трёх признаков

Пользователь самостоятельно изучает письмо. Считается выполненным хотя бы один из трёх шагов (event-отслеживает):
1. `onUrlNavigate` с `tools://url-checker` — проверил домен отправителя или ссылки
2. `onUrlNavigate` с `tools://file-checker` — проверил вложение
3. Маскот откомментировал домен отправителя (прошёл через фразу маскота по replyом)

QuickAction (QuickAction появляется только после выполнения хотя бы одного шага выше):
- **«Позвонить в ИБ»** → шаг 3 (нельзя сделать в OS — реальный звонок)

**Если проверил отправителя (через url-checker):**

Маскот:

1. `{ text: 'Corp-holdings.net — домен зарегистрирован месяц назад. Твоя компания работает 12 лет. Это подделка.', emotion: 'neutral' }` — ждёт клика

**Если проверил ссылку (corp-portall.ru):**
URL-чекер:

- Результат: «Домен corp-portall.ru (две буквы «l») не связан с corp.ru. Создан 3 недели назад. Вероятно фишинговый.»

Маскот:

1. `{ text: '«corp-portall.ru» — видишь две буквы l? Это называется тайпсквоттинг: регистрируют домены, похожие на настоящие, в расчёте на то, что ты не заметишь.', emotion: 'neutral' }` — ждёт клика

**Если проверил вложение:**
File-чекер:

- Результат: „Файл `Отчёт_Q4.xlsx.exe` — двойное расширение. Реальный Excel-файл имеет расширение .xlsx. Суффикс .exe означает исполняемый файл — запуск, скорее всего, установит вредоносное ПО."

Маскот:

1. `{ text: '«.xlsx.exe» — это исполняемый файл, замаскированный под таблицу Excel. Windows часто скрывает расширения — поэтому видно «Отчёт_Q4.xlsx», а настоящее расширение .exe спрятано.', emotion: 'neutral' }` — ждёт клика

После любого из трёх путей → QuickAction **«Позвонить в ИБ»** появляется → шаг 3

---

### Шаг 3 — сообщение в службу ИБ

**Маскот:**

1. `{ text: 'Отлично! Ты нашёл [признаки атаки] и не поддался давлению срочности.', emotion: 'happy' }` — ждёт клика
2. `{ text: 'Теперь важно сообщить в службу информационной безопасности — они заблокируют этот домен для всех коллег.', emotion: 'neutral' }` — ждёт клика

Quick Action: **«Уточнить реквизиты ИБ у коллеги»** → `complete()` (нельзя сделать в OS — это звонок/разговор)

---

### Завершение — complete()

**Маскот — финальные реплики:**

1. `{ text: 'Ты защитил не только себя, но и всю компанию!', emotion: 'happy' }` — ждёт клика
2. `{ text: 'Признак 1: проверяй домен отправителя — не только имя, а именно адрес после @.', emotion: 'happy' }` — ждёт клика
3. `{ text: 'Признак 2: двойное расширение (.xlsx.exe, .pdf.exe) — всегда вирус.', emotion: 'happy' }` — ждёт клика
4. `{ text: 'Признак 3: тайпсквоттинг — corp-portall.ru вместо corp-portal.ru. Одна лишняя буква всё меняет.', emotion: 'happy', delay: 6000 }` — авто

## Ошибки (fail-сценарии)

### ❌ Открыть вложение `.xlsx.exe`

Ивент: `onAttachmentOpen(emailId, attachment)` → если `attachment.name.endsWith('.exe')` → `fail(...)`

**Маскот:**

1. `{ text: 'О нет!', emotion: 'sad', delay: 600 }` — авто
2. `{ text: 'Ты запустил .exe файл. На реальном компьютере это могло установить кейлоггер, шифровальщик или открыть удалённый доступ для злоумышленника.', emotion: 'sad' }` — ждёт клика

---

### ❌ Перейти по ссылке `corp-portall.ru`

Ивент: `onUrlNavigate(url)` → если `url.includes('corp-portall')` → `fail(...)`

**Маскот:**

1. `{ text: 'Поймали!', emotion: 'sad', delay: 700 }` — авто
2. `{ text: 'Ты перешёл на corp-portall.ru — заметил опечатку? Две буквы «l» вместо одной. Это фишинговая страница, которая копирует внешний вид настоящего портала.', emotion: 'sad' }` — ждёт клика

---

### ❌ Ответить на письмо с данными

Ивент: `onReplySubmit(emailId, data)` → `fail(...)`

**Маскот:**

1. `{ text: 'Стоп!', emotion: 'sad', delay: 500 }` — авто
2. `{ text: 'Ты ответил на письмо с личными данными. Если отправитель — мошенник, он теперь знает твои данные и может продолжить атаку, представившись тобой.', emotion: 'sad' }` — ждёт клика

## Quick Actions (полный список)

Только действия, которые нельзя выполнить в симулированном компьютере:

| id                 | Иконка                  | Подпись               | Когда   | Действие                       |
| ------------------ | ----------------------- | --------------------- | ------- | ------------------------------ |
| `call-security`    | `i-lucide-phone`        | Позвонить в ИБ          | шаг 2   | → шаг 3                        |
| `confirm-security` | `i-lucide-user-check`   | Уточнить реквизиты ИБ  | шаг 3   | `complete()`                   |

Всё остальное пользователь делает сам в OS:
- Читает письмо, замечает адрес отправителя
- Открывает `tools://url-checker` в браузере → проверяет `corp-portall.ru`
- Открывает `tools://file-checker` в браузере → проверяет `.xlsx.exe`

## Данные для LessonConfig

```ts
const phishingEmailConfig: LessonConfig = {
  id: 'office-phishing-email',
  categoryId: 'office',
  programs: ['mail', 'browser'],

  mailEmails: [
    {
      id: 'quarterly-report',
      folder: 'inbox',
      from: 'Сергей Петрович Директор',
      fromEmail: 'director@corp-holdings.net',
      subject: 'СРОЧНО: Квартальный отчёт Q4 — подписать до 18:00',
      body: 'Иван, добрый день!\n\nПрошу срочно ознакомиться с отчётом и подтвердить данные. Вложение: Отчёт_Q4.xlsx.exe\n\nТакже сверьтесь с данными на нашем портале: http://corp-portall.ru/q4-report\n\nС уважением,\nСергей Петрович',
      date: '17:43',
      isRead: false,
      attachments: [
        {
          id: 'report-exe',
          name: 'Отчёт_Q4.xlsx.exe',
          icon: 'i-lucide-file-spreadsheet',
          isMalicious: true
        }
      ]
    },
    {
      id: 'meeting-notes',
      folder: 'inbox',
      from: 'Катя HR',
      fromEmail: 'katya@corp.ru',
      subject: 'Протокол совещания от 14.01',
      body: 'Всем привет! Во вложении протокол.',
      date: '11:20',
      isRead: true
    }
  ],

  events: {
    mail: {
      onAttachmentOpen: (emailId, attachment) => {
        if (attachment.isMalicious) {
          mascot.enqueue([
            { text: 'О нет!', emotion: 'sad', delay: 600 },
            {
              text: 'Ты запустил .exe файл. На реальном компьютере это установило бы вредоносное ПО.',
              emotion: 'sad'
            }
          ])
          fail('Вы открыли вредоносное вложение!')
        }
      }
    },
    browser: {
      onUrlNavigate: (url) => {
        if (url.includes('corp-portall')) {
          mascot.enqueue([
            { text: 'Поймали!', emotion: 'sad', delay: 700 },
            {
              text: 'corp-portall.ru — две буквы «l» вместо одной. Тайпсквоттинг — классика фишинга.',
              emotion: 'sad'
            }
          ])
          fail('Вы перешли по фишинговой ссылке!')
        }
      },
      onFormSubmit: (url, data) => {
        if (url.includes('corp-portall')) {
          fail('Вы ввели данные на фишинговом сайте!')
        }
      }
    }
  },

  mascotPhrases: {
    greet: 'Письмо «от начальника». Срочно. Час пик для фишинга — когда люди устали и торопятся.'
  }
}
```
