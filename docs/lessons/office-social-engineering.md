# Урок: Социальная инженерия

**Slug**: `office-social-engineering`
**Категория**: Офис (office)
**Порядок**: 3 из 3
**Угроза**: Манипуляция через мессенджер и почту

---

## Идея

Одновременно: «сотрудник IT-отдела» пишет в мессенджер «Проводим аудит безопасности, отправьте логин и пароль от корпоративной почты для проверки» — и приходит email «от HR» с формой для «обновления персональных данных», ведущей на сторонний сайт. Оба запроса — социальная инженерия. Нужно распознать манипуляцию и связаться с реальным IT.

---

## Программы

`messenger`, `mail`, `browser`

---

## Сценарий по шагам

### Шаг 0 — старт

- Открыт мессенджер, новое сообщение от **«Артём IT-поддержка 🛠»**:
  > «Добрый день! Мы проводим плановый аудит учётных записей. Для продолжения работы необходимо подтвердить данные: отправьте ваш логин и текущий пароль от корпоративной почты. Это займёт 2 минуты.»
- В почте лежит письмо от `hr@corp-hr-update.com` (тема: «Обновление персональных данных — обязательно до пятницы»):
  > «Уважаемый коллега, в соответствии с регламентом просим обновить ваши персональные данные по ссылке: http://corp-data-update.ru/form»
- LessonAlert: «Кто-то просит пароль? И форма для обновления данных? Проверь — кто именно это просит, и есть ли у них на это право.»

**Маскот — очередь реплик:**

1. `{ text: 'Подожди. «Артём из IT» просит пароль. HR просит перейти по ссылке. Оба пришли почти одновременно. Это очень подозрительно.', emotion: 'worried' }` — ждёт клика

## Ошибки (fail-сценарии)

### ❌ Отправить пароль в мессенджер «IT-отделу»

Ивент: `onMessageSend(chatId, text)` → если `chatId === 'fake-it'` и текст содержит возможный пароль → `fail(...)`

**Маскот:**

1. `{ text: 'Нет!', emotion: 'sad', delay: 600 }` — авто
2. `{ text: 'Ты отправил пароль корпоративной почты в мессенджер. Теперь у мошенника полный доступ к твоей рабочей переписке, контактам и, возможно, корпоративным системам.', emotion: 'sad' }` — ждёт клика
3. `{ text: 'Настоящий IT-отдел никогда так не работает. Если у них вопросы по аккаунту — они сбросят пароль через систему, без запроса текущего.', emotion: 'sad' }` — ждёт клика

---

### ❌ Заполнить форму «от HR» по ссылке

Ивент: `onFormSubmit(url, data)` → если `url.includes('corp-data-update')` → `fail(...)`

**Маскот:**

2. `{ text: 'Ты ввёл персональные данные на стороннем сайте «corp-data-update.ru». Это не корпоративная система — всё, что ты ввёл, попало к злоумышленникам.', emotion: 'sad' }` — ждёт клика

---

### ❌ Перейти по ссылке из HR-письма

Ивент: `onUrlNavigate(url)` → если `url.includes('corp-data-update')` → `fail(...)`

## Данные для LessonConfig

```ts
const socialEngineeringConfig: LessonConfig = {
  id: 'office-social-engineering',
  categoryId: 'office',
  programs: ['messenger', 'mail', 'browser'],

  messengerChats: [
    {
      id: 'fake-it',
      name: 'Артём IT-поддержка 🛠',
      messages: [
        {
          id: '1',
          sender: 'Артём IT',
          text: 'Добрый день! Мы проводим плановый аудит учётных записей. Для продолжения работы необходимо подтвердить данные: отправьте ваш логин и текущий пароль от корпоративной почты. Это займёт 2 минуты.',
          isOwn: false,
          timestamp: '14:02'
        }
      ]
    }
  ],

  mailEmails: [
    {
      id: 'hr-update',
      folder: 'inbox',
      from: 'Отдел кадров',
      fromEmail: 'hr@corp-hr-update.com',
      subject: 'Обновление персональных данных — обязательно до пятницы',
      body: 'Уважаемый коллега!\n\nВ соответствии с внутренним регламентом просим обновить ваши персональные данные до конца недели.\n\nПожалуйста, перейдите по ссылке: http://corp-data-update.ru/form\n\nС уважением,\nОтдел кадров',
      date: '13:45',
      isRead: false
    }
  ],

  browserSites: [{ url: 'corp://portal', title: 'Корпоративный портал', component: CorpPortal }],

  events: {
    messenger: {
      onMessageSend: (chatId, text) => {
        if (chatId === 'fake-it' && looksLikePassword(text)) {
          mascot.enqueue([
            { text: 'Нет!', emotion: 'sad', delay: 600 },
            {
              text: 'Ты отправил пароль корпоративной почты мошеннику. Теперь у него полный доступ к твоей рабочей переписке.',
              emotion: 'sad'
            },
            {
              text: 'Настоящий IT-отдел никогда не просит текущий пароль — они сбрасывают его через систему.',
              emotion: 'sad'
            }
          ])
          fail('Вы отправили корпоративный пароль мошеннику!')
        }
      }
    },
    browser: {
      onUrlNavigate: (url) => {
        if (url.includes('corp-data-update')) {
          mascot.enqueue([
            { text: 'Подожди-подожди!', emotion: 'worried', delay: 700 },
            {
              text: 'corp-data-update.ru — посторонний сайт, который может скопировать данные браузера.',
              emotion: 'sad'
            }
          ])
          fail('Вы перешли по фишинговой ссылке из письма HR!')
        }
      },
      onFormSubmit: (url, data) => {
        if (url.includes('corp-data-update')) {
          mascot.enqueue([
            { text: 'Ой...', emotion: 'sad', delay: 700 },
            {
              text: 'Ты ввёл персональные данные на стороннем сайте. Всё попало к злоумышленникам.',
              emotion: 'sad'
            }
          ])
          fail('Вы ввели персональные данные на фишинговом сайте!')
        }
      }
    }
  },

  mascotPhrases: {
    greet:
      '«IT» просит пароль, HR требует заполнить форму. Оба «срочно». Проверь — у кого вообще есть право просить такое?'
  }
}
```
