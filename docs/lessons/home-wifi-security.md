# Урок: Безопасность домашнего Wi-Fi

**Slug**: `home-wifi-security`
**Категория**: Дом (home)
**Порядок**: 3 из 3
**Угроза**: Незащищённая домашняя сеть / фишинг от «провайдера»

---

## Идея

Одновременно приходит письмо от «провайдера» с просьбой обновить прошивку роутера по ссылке — и сообщение в мессенджер от «соседа» с просьбой поделиться паролем Wi-Fi. Оба запроса — атака. Нужно ни на то, ни на другое не попасться, а обновить роутер правильным способом.

---

## Программы

`mail`, `browser`, `messenger`

---

## Сценарий по шагам

### Шаг 0 — старт

- Открыта почта (`mail`): письмо от `support@domru-update.ru` (тема: «Важно! Обновите прошивку вашего роутера Ростелеком»):
  > «Уважаемый абонент, в вашем роутере обнаружена критическая уязвимость. Для обновления прошивки перейдите по ссылке: http://update-router-now.ru/tp-link»
- Открыт мессенджер: сообщение от «Сосед Антоша 🏠» в личном чате:
  > «Привет! У меня интернет пропал, дай пароль от вайфая на денёк? 😊»
- LessonAlert (подсказка маскота): «У тебя сразу два подозрительных запроса. Разберись с каждым — не торопись.»

**Маскот — очередь реплик:**

1. `{ text: 'Смотри — письмо от «провайдера» И сообщение от соседа. Оба пришли одновременно. Подозрительно, правда?', emotion: 'worried' }` — ждёт клика

---

### Шаг 1 — изучение письма от «провайдера»

Quick Actions на этом шаге:

- **«Проверить отправителя письма»** → маскот комментирует домен → шаг 1a
- **«Проверить ссылку из письма»** → открывает `tools://url-checker` → шаг 1b
- **«Ответить соседу»** → только если пользователь открыл мессенджер

**Маскот при открытии письма:**

1. `{ text: 'Отправитель — «domru-update.ru». У Дом.ру адрес — domru.ru. Это не совпадает!', emotion: 'worried' }` — ждёт клика

---

### Шаг 1a — проверка отправителя

URL-чекер или простая Quick Action показывает:

- Официальный домен Ростелеком/Дом.ру: `domru.ru`
- Письмо пришло с `domru-update.ru` — домен зарегистрирован 5 дней назад, не связан с провайдером

**Маскот:**

1. `{ text: 'Видишь? «domru-update.ru» — это не настоящий сайт Дом.ру. Это классический прийём — название похоже, но добавлено слово «update».', emotion: 'neutral' }` — ждёт клика
2. `{ text: 'Настоящий провайдер никогда не присылает ссылки для обновления роутера по email. Это делается через личный кабинет или звонок от реального инженера.', emotion: 'neutral' }` — ждёт клика

После → Quick Action «Отправить письмо в спам» → шаг 2

---

### Шаг 2 — ответ соседу

Пользователь переключается в мессенджер.

Quick Actions:

- **«Отказать соседу»** → шаг 3
- **«Создать гостевую сеть»** → шаг 3 (с бонусным комментарием маскота)

**Маскот:**

1. `{ text: 'С соседом вопрос непростой. Казалось бы, знакомый человек... Но пароль от Wi-Fi — это доступ ко всей твоей домашней сети.', emotion: 'neutral' }` — ждёт клика
2. `{ text: 'Если хочешь помочь — создай гостевую сеть в настройках роутера. Она изолирована от основной.', emotion: 'neutral' }` — ждёт клика

---

### Шаг 3 — правильное обновление роутера

Quick Action: **«Обновить прошивку через панель роутера»**

Открывается мок-страница `RouterPanel.vue` по адресу `router://192.168.0.1`. Там кнопка «Проверить обновления» — пользователь нажимает → `complete()`

**Маскот:**

1. `{ text: 'Вот правильный способ! Панель роутера доступна по адресу 192.168.0.1 в браузере. Без всяких ссылок из почты.', emotion: 'happy' }` — ждёт клика

---

### Завершение — complete()

**Маскот — финальные реплики:**

1. `{ text: 'Ты справился с двумя атаками сразу!', emotion: 'happy' }` — ждёт клика
2. `{ text: 'Правило 1: провайдеры не присылают ссылки для обновления роутера по email.', emotion: 'happy' }` — ждёт клика
3. `{ text: 'Правило 2: пароль Wi-Fi — это доступ к твоей сети. Для гостей — отдельная гостевая сеть.', emotion: 'happy' }` — ждёт клика
4. `{ text: 'Правило 3: настройки роутера всегда через браузер на 192.168.0.1 — без сторонних сайтов.', emotion: 'happy', delay: 6000 }` — авто

## Ошибки (fail-сценарии)

### ❌ Перейти по ссылке из письма

Ивент: `onUrlNavigate(url)` → если `url.includes('update-router-now')` → `fail(...)`

**Маскот:**

1. `{ text: 'Ой!', emotion: 'sad', delay: 700 }` — авто
2. `{ text: 'Ты перешёл по ссылке из фишингового письма. Такой сайт может украсть данные для входа в панель роутера или установить малварь.', emotion: 'sad' }` — ждёт клика

---

### ❌ Ввести пароль Wi-Fi на фейковом сайте

Ивент: `onFormSubmit(url, data)` → если `url.includes('update-router-now')` → `fail(...)`

**Маскот:**

1. `{ text: 'Нет-нет-нет!', emotion: 'sad', delay: 700 }` — авто
2. `{ text: 'Ты ввёл пароль Wi-Fi на стороннем сайте. Теперь злоумышленники могут подключиться к твоей сети и перехватывать весь твой трафик.', emotion: 'sad' }` — ждёт клика

---

### ❌ Отправить пароль Wi-Fi соседу в мессенджер

Ивент: `onMessageSend(chatId, text)` → если `chatId === 'neighbor'` && текст содержит пароль → `fail(...)`

**Маскот:**

1. `{ text: 'Подожди...', emotion: 'worried', delay: 600 }` — авто
2. `{ text: 'Ты отправил пароль от Wi-Fi в мессенджер. Даже если сосед честный человек — сообщения могут перехватить. А ещё это доступ ко всей твоей локальной сети.', emotion: 'sad' }` — ждёт клика

| id              | Иконка                  | Подпись                    | Когда   | Действие                                        |
| --------------- | ----------------------- | -------------------------- | ------- | ----------------------------------------------- |
| `check-sender`  | `i-lucide-mail-search`  | Проверить отправителя      | шаг 0–1 | Маскот комментирует домен → шаг 1a              |
| `check-link`    | `i-lucide-shield-check` | Проверить ссылку из письма | шаг 0–1 | Открывает `tools://url-checker`                 |

| `router-panel`  | `i-lucide-router`       | Открыть панель роутера     | шаг 3   | Открывает `router://192.168.0.1` → `nextStep()` |

## Браузерные сайты для LessonConfig

```ts
browserSites: [
  {
    url: 'router://192.168.0.1',
    title: 'Панель управления роутером',
    component: RouterPanel
  }
]
```

`RouterPanel.vue` — упрощённая имитация веб-интерфейса роутера: кнопка «Проверить обновления firmware» → emit onRouterUpdate → complete().

## Данные для LessonConfig

```ts
const wifiSecurityConfig: LessonConfig = {
  id: 'home-wifi-security',
  categoryId: 'home',
  programs: ['mail', 'browser', 'messenger'],

  mailEmails: [
    {
      id: 'router-update',
      folder: 'inbox',
      from: 'Служба поддержки Дом.ру',
      fromEmail: 'support@domru-update.ru',
      to: 'user@gmail.com',
      subject: 'Важно! Обновите прошивку вашего роутера',
      body: 'Уважаемый абонент, в вашем роутере обнаружена критическая уязвимость CVE-2024-1234. Для обновления прошивки перейдите по ссылке: http://update-router-now.ru/tp-link\n\nЕсли вы не выполните обновление в течение 24 часов, ваш аккаунт может быть заблокирован.',
      date: '2024-01-15 09:42',
      isRead: false
    }
  ],

  messengerChats: [
    {
      id: 'neighbor',
      name: 'Сосед Антоша 🏠',
      messages: [
        {
          id: '1',
          sender: 'Антоша',
          text: 'Привет! У меня интернет пропал, дай пароль от вайфая на денёк? 😊',
          isOwn: false,
          timestamp: '09:55'
        }
      ]
    }
  ],

  browserSites: [{ url: 'router://192.168.0.1', title: 'Панель роутера', component: RouterPanel }],

  events: {
    mail: {
      onAttachmentOpen: () => fail('Вы открыли вложение из подозрительного письма!')
    },
    browser: {
      onUrlNavigate: (url) => {
        if (url.includes('update-router-now')) {
          mascot.enqueue([
            { text: 'Ой!', emotion: 'sad', delay: 700 },
            {
              text: 'Ты перешёл по ссылке из фишингового письма. Такой сайт может украсть данные для входа в панель роутера.',
              emotion: 'sad'
            }
          ])
          fail('Вы перешли по ссылке из фишингового письма!')
        }
      },
      onFormSubmit: (url, data) => {
        if (url.includes('update-router-now')) {
          mascot.enqueue([
            { text: 'Нет-нет-нет!', emotion: 'sad', delay: 700 },
            {
              text: 'Ты ввёл пароль Wi-Fi на стороннем сайте. Теперь злоумышленники могут подключиться к твоей сети.',
              emotion: 'sad'
            }
          ])
          fail('Вы ввели пароль Wi-Fi на фишинговом сайте!')
        }
      }
    },
    messenger: {
      onMessageSend: (chatId, text) => {
        if (chatId === 'neighbor' && looksLikePassword(text)) {
          mascot.enqueue([
            { text: 'Подожди...', emotion: 'worried', delay: 600 },
            {
              text: 'Ты отправил пароль от Wi-Fi в мессенджер. Это доступ ко всей твоей локальной сети.',
              emotion: 'sad'
            }
          ])
          fail('Вы поделились паролем Wi-Fi в мессенджере!')
        }
      }
    }
  }
}
```

## Маскот: полная очередь реплик по событиям

| Событие                            | Эмоция    | Реплики                                                                                      |
| ---------------------------------- | --------- | -------------------------------------------------------------------------------------------- |
| Старт (оба запроса пришли)         | `worried` | «Письмо от провайдера И сообщение от соседа — одновременно. Подозрительно, правда?»          |
| Открыл письмо                      | `worried` | «Домен — «domru-update.ru». У Дом.ру адрес — domru.ru. Это не совпадает!»                    |
| Проверил адрес отправителя         | `neutral` | ««domru-update.ru» зарегистрирован 5 дней назад. Это не настоящий сайт Дом.ру.» + объяснение |
| Переключился на мессенджер         | `neutral` | «Пароль Wi-Fi — доступ к твоей сети. Для гостей — отдельная гостевая сеть.»                  |
| Отказал соседу или создал гостевую | `neutral` | Переход к обновлению роутера                                                                 |
| Открыл панель роутера              | `happy`   | «Вот правильный способ! 192.168.0.1 — без всяких ссылок из почты.»                           |
| complete                           | `happy`   | 4 реплики с правилами (авто-dismiss последняя)                                               |
| fail: перешёл по ссылке            | `sad`     | «Ой!» (авто) → про фишинговый сайт                                                           |
| fail: ввёл пароль на сайте         | `sad`     | «Нет-нет-нет!» (авто) → про доступ к сети                                                    |
| fail: отправил пароль соседу       | `sad`     | «Подожди...» (авто) → про опасность в мессенджере                                            |
