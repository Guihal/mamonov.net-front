# Задача 1.8: Создание программы Mail

**Статус**: ⬜ Не выполнено

## Описание

Создать программу "Почта" — визуально похожа на Apple Mail / Outlook. Слева — список писем, справа — просмотр выбранного. Общие компоненты выносятся в программу, данные и ивенты инжектируются уроком.

## Архитектура

```
OS/Window (общий)
└── Content → Mail/index.vue
    ├── MailSidebar     ← папки (Входящие, Отправленные, Спам)
    ├── MailList        ← список писем в выбранной папке
    └── MailView        ← просмотр письма
        ├── MailHeader  ← от, кому, тема, дата
        ├── MailBody    ← тело письма (HTML-мок)
        └── Attachments ← вложения (если есть)
```

### Данные (инжектируются уроком)

```ts
interface MailConfig {
  folders?: MailFolder[] // доп. папки (урок расширяет)
  emails: EmailConfig[] // письма
  events?: MailEvents
}

interface MailFolder {
  id: string
  name: string // «Входящие», «Спам», ...
  icon?: string // lucide иконка
}

interface EmailConfig {
  id: string
  folder: string // id папки
  from: string // имя отправителя
  fromEmail: string // email отправителя
  to?: string
  subject: string
  body: string // HTML контент
  date: string
  isRead?: boolean
  attachments?: AttachmentConfig[]
}

interface AttachmentConfig {
  id: string
  name: string // «report.xlsx»
  icon?: string
  isMalicious?: boolean // для gameController
}

interface MailEvents {
  onMailOpen?: (emailId: string) => void
  onAttachmentOpen?: (emailId: string, attachmentId: string) => void
  onLinkClick?: (emailId: string, url: string) => void
  onReply?: (emailId: string, text: string) => void
}
```

#### 1.8.1 — `useMailState.ts` (`app/composables/os/`)

Единый composable для состояния почты. По аналогии с `useExplorerState.ts` — принимает `MailEvents`, возвращает реактивное состояние + методы.

```ts
// app/composables/os/useMailState.ts
import type { MailFolder, EmailConfig, MailEvents } from '~/types/programs'

const BASE_FOLDERS: MailFolder[] = [

export function useMailState(events?: MailEvents) {
  const folders = ref<MailFolder[]>([...BASE_FOLDERS])
  const emails = ref<EmailConfig[]>(deepClone(BASE_EMAILS))

  /** ID текущей активной папки */
  const activeFolderId = ref<string>('inbox')

  /** ID текущего открытого письма (null = ничего не выбрано) */
  const activeEmailId = ref<string | null>(null)

  /** Текущая папка */
  const activeFolder = computed(
    () => folders.value.find((f) => f.id === activeFolderId.value) ?? folders.value[0]
  )

  /** Письма в текущей папке, отсортированные по дате (новые сверху) */
  const folderEmails = computed(() =>
    emails.value
      .filter((e) => e.folder === activeFolderId.value)
      .sort((a, b) => new Date(b.date).getTime() - new Date(a.date).getTime())
  )

  /** Текущее открытое письмо */
  const activeEmail = computed(() =>
    activeEmailId.value ? (emails.value.find((e) => e.id === activeEmailId.value) ?? null) : null
  )

  /** Количество непрочитанных писем по папкам */
  const unreadCounts = computed(() => {
    const counts: Record<string, number> = {}
    for (const folder of folders.value) {
      counts[folder.id] = emails.value.filter((e) => e.folder === folder.id && !e.isRead).length
    }
    return counts
  })

  /** Переключить активную папку */
  const setActiveFolder = (folderId: string) => {
    activeFolderId.value = folderId
    activeEmailId.value = null // сбрасываем выбор при смене папки
  }

  /** Открыть письмо */
  const openEmail = (emailId: string) => {
    activeEmailId.value = emailId
    // Помечаем как прочитанное
    const email = emails.value.find((e) => e.id === emailId)
    if (email && !email.isRead) {
      email.isRead = true
    }
    events?.onMailOpen?.(emailId)
  }

  /** Обработка клика по ссылке в теле письма */
  const handleLinkClick = (url: string) => {
    if (!activeEmailId.value) return
    events?.onLinkClick?.(activeEmailId.value, url)
  }

  /** Обработка открытия вложения */
  const handleAttachmentOpen = (attachmentId: string) => {
    if (!activeEmailId.value) return
    events?.onAttachmentOpen?.(activeEmailId.value, attachmentId)
  }

  /** Обработка ответа на письмо */
  const handleReply = (text: string) => {
    if (!activeEmailId.value) return
    events?.onReply?.(activeEmailId.value, text)
  }

  /** Урок добавляет письма */
  const addEmails = (newEmails: EmailConfig[]) => {
    emails.value.push(...deepClone(newEmails))
  }

  /** Урок добавляет папки */
  const addFolders = (newFolders: MailFolder[]) => {
    for (const folder of newFolders) {
      // Не дублируем, если папка с таким id уже есть
      if (!folders.value.find((f) => f.id === folder.id)) {
        folders.value.push(folder)
      }
    }
  }

  return {
    folders: readonly(folders),
    emails: readonly(emails),
    activeFolderId: readonly(activeFolderId),
    activeFolder,
    activeEmail,
    activeEmailId: readonly(activeEmailId),
    folderEmails,
    unreadCounts,
    setActiveFolder,
    openEmail,
    handleLinkClick,
    handleAttachmentOpen,
    handleReply,
    addEmails,
    addFolders
  }
}

function deepClone<T>(obj: T): T {
  return JSON.parse(JSON.stringify(obj))
}
```

API composable:

| Метод / Свойство       | Тип                                   | Описание                                    |
| ---------------------- | ------------------------------------- | ------------------------------------------- |
| `folders`              | `Readonly<Ref<MailFolder[]>>`         | Все папки (базовые + урок)                  |
| `emails`               | `Readonly<Ref<EmailConfig[]>>`        | Все письма (базовые + урок)                 |
| `activeFolderId`       | `Readonly<Ref<string>>`               | ID текущей папки                            |
| `activeFolder`         | `ComputedRef<MailFolder>`             | Текущая папка                               |
| `activeEmail`          | `ComputedRef<EmailConfig \| null>`    | Текущее письмо                              |
| `activeEmailId`        | `Readonly<Ref<string \| null>>`       | ID текущего письма                          |
| `folderEmails`         | `ComputedRef<EmailConfig[]>`          | Письма в текущей папке (сортировка по дате) |
| `unreadCounts`         | `ComputedRef<Record<string, number>>` | Кол-во непрочитанных по папкам              |
| `setActiveFolder`      | `(folderId: string) => void`          | Переключить папку (сбрасывает выбор)        |
| `openEmail`            | `(emailId: string) => void`           | Открыть письмо + `markRead` + `onMailOpen`  |
| `handleLinkClick`      | `(url: string) => void`               | Клик по ссылке → `onLinkClick`              |
| `handleAttachmentOpen` | `(attachmentId: string) => void`      | Открыть вложение → `onAttachmentOpen`       |
| `handleReply`          | `(text: string) => void`              | Ответить → `onReply`                        |
| `addEmails`            | `(emails: EmailConfig[]) => void`     | Добавить письма из урока                    |
| `addFolders`           | `(folders: MailFolder[]) => void`     | Добавить папки из урока                     |

#### 1.8.9 — `MailMessage.vue`

Элемент письма в списке (clickable row).

```vue
<script setup lang="ts">
import type { EmailConfig } from '~/types/programs'

defineProps<{
  email: EmailConfig
  isActive: boolean
}>()

defineEmits<{
  click: []
}>()

/** Форматируем дату: сегодня → «12:30», иначе → «3 апр» */
const formatDate = (dateStr: string): string => {
  const date = new Date(dateStr)
  const now = new Date()
  const isToday =
    date.getDate() === now.getDate() &&
    date.getMonth() === now.getMonth() &&
    date.getFullYear() === now.getFullYear()

  if (isToday) {
    return date.toLocaleTimeString('ru-RU', { hour: '2-digit', minute: '2-digit' })
  }
  return date.toLocaleDateString('ru-RU', { day: 'numeric', month: 'short' })
}

/** Превью: первые ~80 символов тела (без HTML тегов) */
const preview = (body: string): string => {
  const text = body
    .replace(/<[^>]+>/g, '')
    .replace(/\s+/g, ' ')
    .trim()
  return text.length > 80 ? text.slice(0, 80) + '…' : text
}
</script>

<template>
  <button
    class="mail-message"
    :class="{
      'mail-message--active': isActive,
      'mail-message--unread': !email.isRead
    }"
    @click="$emit('click')"
  >
    <div class="mail-message__top">
      <span class="mail-message__from">{{ email.from }}</span>
      <span class="mail-message__date">{{ formatDate(email.date) }}</span>
    </div>
    <div class="mail-message__subject">{{ email.subject }}</div>
    <div class="mail-message__preview">{{ preview(email.body) }}</div>
    <div v-if="email.attachments?.length" class="mail-message__attachment-indicator">

    </div>

</template>
```

#### 1.8.8 — `MailView.vue`

Правая колонка — просмотр выбранного письма. Если письмо не выбрано — placeholder.

```vue
<script setup lang="ts">
const mailState = inject('mailState') as ReturnType<typeof useMailState>
const { activeEmail, handleLinkClick, handleAttachmentOpen } = mailState

/** Форматируем полную дату: «3 апреля 2026, 14:30» */
const formatFullDate = (dateStr: string): string => {
  const date = new Date(dateStr)
  return date.toLocaleDateString('ru-RU', {
    day: 'numeric',
    month: 'long',
    year: 'numeric',
    hour: '2-digit',
    minute: '2-digit'
  })
}

/**
 * Перехват кликов по ссылкам в body письма.
 * Вместо настоящего перехода — вызываем handleLinkClick.
 */
const onBodyClick = (e: MouseEvent) => {
  const target = e.target as HTMLElement
  const anchor = target.closest('a')
  if (anchor) {
    e.preventDefault()
    const url = anchor.getAttribute('href')
    if (url) handleLinkClick(url)
  }
}
</script>

<template>
  <div class="mail-view">
    <template v-if="activeEmail">
```

### Фаза 3: Интеграция

#### 1.8.11 — Ивенты

Ивенты инжектируются из `lessonConfig.events.mail` (тип `MailEvents`):

- `openEmail()` → `onMailOpen?.(emailId)` — вызывается при клике на письмо в списке
- `handleAttachmentOpen()` → `onAttachmentOpen?.(emailId, attachmentId)` — вызывается при клике по вложению
- `onBodyClick()` → `onLinkClick?.(emailId, url)` — вызывается при клике на ссылку в теле письма
- `handleReply()` → `onReply?.(emailId, text)` — вызывается при отправке ответа (если реализован Reply)

GameController подписывается на эти ивенты и проверяет правильность действий игрока:

```ts
// Пример в конфиге урока:
events: {
  mail: {
    onAttachmentOpen: (emailId, attachmentId) => {
      // Если это фишинговое вложение — fail
      gameController.fail('Вы открыли вредоносное вложение!')
    },
    onLinkClick: (emailId, url) => {
      // Если это фишинговая ссылка — fail
      if (url.includes('phishing')) {
        gameController.fail('Вы перешли по фишинговой ссылке!')
      }
    }
  }
}
```
