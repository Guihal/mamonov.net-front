# Задача 1.7: Создание программы Browser

**Статус**: ⬜ Не выполнено

## Описание

Создать программу "Браузер" — внутри стандартного OS-окна, под его header'ом. Визуально имитирует Chrome. Вкладки живут внутри content-зоны окна (не трогаем header окна — он общий для всех программ). Неизвестные URL открываются в `<iframe>`, известные рендерят кастомный Vue-компонент.

## Архитектура

```
OS/Window (общий, header + traffic lights)
└── Content → Browser/index.vue
    ├── TabBar          ← вкладки (Chrome-style)
    ├── AddressBar      ← адресная строка + кнопки
    ├── BookmarksBar    ← закладки (базовые + урок расширяет)
    └── PageView        ← контент: кастомный компонент ИЛИ iframe
```

### Вкладки (TabBar)

- Несколько вкладок в одном окне браузера
- Каждая вкладка хранит `{ url: string, title: string, component?: Component }`
- Если URL есть в реестре известных сайтов → `title` = название сайта
- Если URL **нет** в реестре → `title` = сам URL (как в реальном Chrome, пока не загрузится)
- Кнопка `+` для новой вкладки, `×` для закрытия

### Реестр известных сайтов (расширяемый)

Базовые сайты (всегда доступны):

| Сайт                 | URL (внутренний)       | Компонент         | Описание                                  |
| -------------------- | ---------------------- | ----------------- | ----------------------------------------- |
| Поисковик            | `search://home`        | `SearchPage.vue`  | Google-like мок, инпут → результаты (мок) |
| Проверка URL         | `tools://url-checker`  | `UrlChecker.vue`  | Инпут URL → «безопасен / опасен» (мок)    |
| Проверка вложений    | `tools://file-checker` | `FileChecker.vue` | Загрузить файл → результат проверки (мок) |
| Корпоративный портал | `corp://portal`        | `CorpPortal.vue`  | Инфо о компании, объявления (мок)         |

Урок расширяет реестр через конфиг:

```ts
interface BrowserSiteConfig {
  url: string // внутренний URL
  title: string // заголовок вкладки
  component: Component // Vue-компонент страницы
  favicon?: string // иконка (для вкладки + bookmarks)
}

// В конфиге урока:
lessonConfig.browserSites = [
  { url: 'phishing://fake-bank', title: 'ВашБанк', component: FakeBankPage }
]
```

### Неизвестные URL

Если URL не найден в реестре → `<iframe :src="url" sandbox="allow-scripts allow-same-origin" />`. Это позволяет открывать реальные сайты (но с ограничениями sandbox).

### Адресная строка (AddressBar)

- `UInput` с `variant="outline"`, `rounded-full`, `bg-gray-100`
- Показывает URL текущей вкладки
- Ввод URL + Enter → навигация (поиск в реестре → компонент или iframe)
- Кнопки: ← (назад), → (вперёд), ↻ (обновить) — `UButton variant="ghost" size="xs"`
- Иконка ★ (избранное) — декоративная

### Панель закладок (BookmarksBar)

Базовые закладки (всегда):

| Закладка             | URL                    | Иконка                  |
| -------------------- | ---------------------- | ----------------------- |
| Проверка URL         | `tools://url-checker`  | `i-lucide-shield-check` |
| Проверка файлов      | `tools://file-checker` | `i-lucide-file-search`  |
| Корпоративный портал | `corp://portal`        | `i-lucide-building`     |

Урок добавляет свои:

```ts
lessonConfig.browserBookmarks = [
  { label: 'URLScan', url: 'https://urlscan.io', favicon: '/icons/urlscan.png' }
]
```

### Ивенты (инжектируются уроком через gameController)

```ts
interface BrowserEvents {
  onUrlNavigate?: (url: string) => void
  onFormSubmit?: (url: string, data: Record<string, string>) => void
  onBookmarkClick?: (url: string) => void
  onTabOpen?: (url: string) => void
  onTabClose?: (url: string) => void
}
```

## Детальный план реализации

### Фаза 1: Состояние и данные (composables)

#### 1.7.1 — `useBrowserSites.ts` (`app/composables/os/`)

Реестр известных сайтов. По аналогии с `useExplorerState.ts` — composable, не глобальный state.

```ts
// API:
// baseSites: Map<string, BrowserSiteConfig> — 4 базовых сайта
// resolve(url: string): BrowserSiteConfig | null — поиск по URL
// registerSites(sites: BrowserSiteConfig[]) — добавление из LessonConfig.browserSites
// readonly allSites — для отладки
```

Конфиг урока инжектируется через `inject('lessonConfig')` → `lessonConfig.browserSites` добавляются в `registerSites()` при mount.

#### 1.7.2 — `useBrowserTabs.ts` (`app/composables/os/`)

Состояние вкладок внутри одного окна браузера:

```ts
interface BrowserTab {
  id: string // nanoid или инкремент
  url: string // текущий URL
  title: string // из реестра или сам URL
  favicon?: string
  history: string[] // стек URL для назад/вперёд
  historyIndex: number // текущая позиция в стеке
}
```

API:

- `tabs: Ref<BrowserTab[]>` — реактивный массив вкладок
- `activeTabId: Ref<string>` — ID активной вкладки
- `activeTab: ComputedRef<BrowserTab>` — текущая вкладка
- `openTab(url?: string)` — новая вкладка (по умолчанию `search://home`), вызывает `onTabOpen`
- `closeTab(id: string)` — закрыть вкладку, вызывает `onTabClose`. Если последняя — закрыть окно
- `setActiveTab(id: string)` — переключить
- `navigateTo(url: string)` — навигация в активной вкладке, обновляет history, вызывает `onUrlNavigate`
- `goBack()` / `goForward()` — по history
- `canGoBack` / `canGoForward` — computed

Принимает параметром `BrowserEvents` (из inject или props) для вызова колбэков.

#### 1.7.3 — `useBrowserBookmarks.ts` (`app/composables/os/`)

```ts
// API:
// baseBookmarks: BookmarkConfig[] — 3 базовых (url-checker, file-checker, corp-portal)
// allBookmarks: ComputedRef<BookmarkConfig[]> — base + lesson
// addBookmarks(items: BookmarkConfig[]) — из LessonConfig.browserBookmarks
```

#### 1.7.4 — `index.vue` (корневой)

По аналогии с Explorer — получает конфиг через `inject`:

```vue
<script setup>
const lessonConfig = inject('lessonConfig', null)
const events = lessonConfig?.events?.browser ?? {}

const sites = useBrowserSites()
const tabs = useBrowserTabs(sites, events)
const bookmarks = useBrowserBookmarks()

// Регистрация сайтов/закладок из урока
if (lessonConfig?.browserSites) sites.registerSites(lessonConfig.browserSites)
if (lessonConfig?.browserBookmarks) bookmarks.addBookmarks(lessonConfig.browserBookmarks)

// provide для дочерних компонентов
provide('browserTabs', tabs)
provide('browserSites', sites)
provide('browserBookmarks', bookmarks)
</script>

<template>
  <div class="browser">
    <BrowserTabBar />
    <BrowserAddressBar />
    <BrowserBookmarksBar />
    <BrowserPageView />
  </div>
</template>
```

Layout: `flex-col`, `h-full`, `overflow-hidden`.

#### 1.7.5 — `TabBar.vue`

- Горизонтальный ряд вкладок, `overflow-x: auto`
- Каждая вкладка: кастомный `<button>` (не UButton — слишком мелкий)
- Активная: `bg-white rounded-[6px]`, неактивная: transparent
- Favicon 16×16 (если есть) + title (truncate) + кнопка `×`
- Кнопка `+` справа
- **Стиль**: контейнер `bg-[#dfe1e5] h-9 border-b border-[#dadce0]`, вкладка `h-[30px] text-[10px] font-medium text-[#3d4043]`

#### 1.7.6 — `AddressBar.vue`

- `UInput variant="outline" size="sm"` с `rounded-full bg-gray-100`
- `v-model` = `activeTab.url`
- `@keydown.enter` → `tabs.navigateTo(inputValue)`
- Слева: кнопки ← → ↻ — `UButton variant="ghost" size="xs"` с `disabled` по `canGoBack/canGoForward`
- Справа: иконка ★ (декоративная)

#### 1.7.7 — `BookmarksBar.vue`

- `v-for` по `allBookmarks`
- Кастомные `<button>`: favicon `<img>` 16×16 + label
- `@click` → `tabs.navigateTo(bookmark.url)` + вызов `onBookmarkClick`
- **Стиль**: `text-xs gap-1`, тонкая `border-b`

#### 1.7.8 — `PageView.vue`

- `computed`: `resolvedSite = sites.resolve(activeTab.url)`
- Если `resolvedSite` → `<component :is="resolvedSite.component" />`
- Иначе → `<iframe :src="url" sandbox="allow-scripts allow-same-origin" class="w-full h-full border-0" />`
- Контейнер: `flex-1 min-h-0 overflow-hidden bg-white`

#### 1.7.9 — `SearchPage.vue` (Google-like)

### Фаза 4: Интеграция

#### 1.7.13 — Ивенты

- `BrowserEvents` инжектируются из `lessonConfig.events.browser`
- `useBrowserTabs` вызывает соответствующие колбэки:
  - `navigateTo()` → `onUrlNavigate?.(url)`
  - `openTab()` → `onTabOpen?.(url)`
  - `closeTab()` → `onTabClose?.(url)`
- `onFormSubmit` вызывается из страниц (SearchPage, UrlChecker) через `inject('browserEvents')`

#### 1.7.14 — Регистрация в PROGRAMS.ts

В `app/utils/PROGRAMS.ts` у `browser` уже нет `component`. Добавить:

```ts
browser: {
```
