# Задача 1.23: Persist Store (фейковая БД)

**Статус**: ✅ Выполнено

## Описание

Пока бекенд не готов, вся «серверная» логика живёт в Pinia-сторе с persist в `localStorage`. Один стор `useMockDbStore` заменяет API — хранит категории, уроки, HP, прогресс. Все текущие `mock*.ts` функции переадресуются на него.

При первом запуске стор инициализируется seed-данными. При повторных визитах — читает сохранённое состояние.

## Связи

| Кто                | Роль                                                       |
| ------------------ | ---------------------------------------------------------- |
| useUserStore       | Вызывает `syncHp`, `hitHp`, `syncProgress` → читает mockDb |

| Страница категорий | Читает список категорий и уроков из mockDb                 |

## Структура стора

```ts
// app/stores/useMockDbStore.ts

interface MockDbState {
  categories: MockCategory[]
  lessons: Record<string, MockLesson[]> // categoryId → lessons
  hp: Record<string, { current: number; max: number; restoredAt: string | null }>
  progress: LessonProgress[]
}

interface MockCategory {
  id: string
  slug: string
  name: string
  description: string
  icon: string // lucide icon name
  difficulty: 1 | 2 | 3
  order: number
}

interface MockLesson {
  id: string
  slug: string
  categoryId: string
  title: string
  description: string
  order: number // порядок внутри категории (0, 1, 2)
  isCompleted: boolean
}
```

## Логика разблокировки уроков

Урок доступен (`isUnlocked`), если:

- `order === 0` (первый в категории) — всегда доступен
- `order > 0` — предыдущий урок (с `order - 1`) имеет статус `isCompleted === true`

Геттер в сторе:

```ts
function isLessonUnlocked(categoryId: string, lessonSlug: string): boolean
```

## Persist

Использовать `pinia-plugin-persistedstate` (если установлен) или ручной `localStorage`:

```ts
export const useMockDbStore = defineStore(
  'mockDb',
  () => {
    // ...state...
  },
  {
    persist: true // если плагин есть
  }
)
```

Если плагина нет — `watch` + `localStorage.setItem` / `onMounted` + `localStorage.getItem`.
