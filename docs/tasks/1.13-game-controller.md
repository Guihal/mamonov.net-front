# Задача 1.13: Создание useGameController

**Статус**: ⬜ Не выполнено

## Назначение

`useGameController` — **центральный нерв всей игровой системы**. Принимает декларативный конфиг урока (`LessonConfig`), поднимает реактивное состояние игры и раздаёт его вниз через Vue `provide`/`inject`. Программы (Browser, Messenger, Mail, Explorer) сами ничего не знают об уроке — они просто вызывают коллбэки, которые gameController передаёт через provide.

Важно понимать разделение ответственности:

| Слой         | Кто                          | Что делает                                                      |
| ------------ | ---------------------------- | --------------------------------------------------------------- |
| Конфиг       | `LessonConfig` (автор урока) | Описывает данные и обработчики событий                          |
| Контроллер   | `useGameController`          | Держит `step`, маршрутизирует `fail`/`complete`, раздаёт данные |
| Хранилище HP | `useUserStore`               | Списывает HP, возвращает актуальный `HpHitResponse`             |
| Маскот       | `useMascotStore`             | Меняет эмоцию, произносит реплики                               |
| GameOver     | `useGameOver`                | Показывает overlay Game Over                                    |
| Программы    | Browser/Mail/Messenger       | Вызывают ивенты из inject, ничего не знают об уроке             |

## Архитектура потоков

```
pages/app/[category]/[lesson]/index.vue
│
├── useGameController(config)          ← точка входа
│   │
│   ├── state
│   │   ├── step: Ref<number>          ← текущий шаг (0-based)
│   │   ├── isCompleted: Ref<boolean>  ← урок пройден
│   │   └── isFailed: Ref<boolean>     ← заблокировано (game over активен)
│   │
│   ├── methods
│   │   ├── nextStep()                 ← правильное действие пользователя
│   │   ├── fail(reason: string)       ← неверное действие
│   │   └── complete()                 ← урок пройден
│   │
│   ├── provide('gameController', ctx) ← вся контекстная инфа
│   ├── provide('lessonConfig', config)← сырой конфиг (для программ)
│   ├── provide('browserEvents', ...)  ← ивенты для Browser
│   ├── provide('messengerEvents', ...)← ивенты для Messenger
│   ├── provide('mailEvents', ...)     ← ивенты для Mail
│   └── provide('explorerEvents', ...)← ивенты для Explorer
│
├── Taskbar: setPinnedPrograms(config.programs)
│
├── OS/Programs/Browser/index.vue
│   └── inject('browserEvents') → useBrowserTabs(events) → onUrlNavigate(url)
│
├── OS/Programs/Messenger/index.vue
│   └── inject('messengerEvents') → useMessengerState(events) → onLinkClick(...)
│
├── OS/Programs/Mail/index.vue
│   └── inject('mailEvents') → useMailState(events) → onAttachmentOpen(...)
│
└── OS/Programs/Explorer/index.vue
    └── inject('explorerEvents') → useExplorerState(events)
```

### Почему `provide`/`inject`, а не Pinia?

Данные урока живут ровно одну страницу урока. При уходе с `[lesson]` всё должно автоматически удалиться. Store Pinia персистентен между маршрутами — пришлось бы чистить вручную. `provide` на уровне компонента страницы даёт автоматический scope: unmount страницы → inject недоступен → нет утечек.

## Тип контекста — `app/types/game.ts`

Создать новый файл. Это то, что раздаётся через `provide('gameController')` и принимается через `inject` в любом компоненте урока.

```ts
// app/types/game.ts
import type { LessonConfig } from './lesson'

export interface GameControllerContext {
  /** Текущий шаг урока (0-based). Реактивный ref. */
  step: Readonly<Ref<number>>
  /** Сколько шагов всего. Вычисляется при старте. */
  totalSteps: number
  /** Урок завершён (правильно). */
  isCompleted: Readonly<Ref<boolean>>
  /** Заблокирован: game over показан. */
  isFailed: Readonly<Ref<boolean>>
  /** Перейти к следующему шагу. Если это последний — вызывает complete(). */
  nextStep: () => void
  /** Пользователь совершил ошибку. Reason — человекочитаемое объяснение. */
  fail: (reason: string) => void
  /** Принудительно завершить урок (правильно). */
  complete: () => void
  /** Сброс всего состояния (для кнопки «Попробовать снова»). */
  reset: () => void
  /** Конфиг урока (read-only, для компонентов, которым нужны данные). */
  config: LessonConfig
}
```

**Ключ provide/inject:** использовать `InjectionKey` для типобезопасности:

```ts
// app/types/game.ts (добавить)
import { type InjectionKey } from 'vue'

export const GAME_CONTROLLER_KEY: InjectionKey<GameControllerContext> = Symbol('gameController')
```

## Реализация — `app/composables/game/useGameController.ts`

```ts
import type { LessonConfig } from '~/types/lesson'
import type { GameControllerContext } from '~/types/game'
import { GAME_CONTROLLER_KEY } from '~/types/game'
import type { BrowserEvents, MessengerEvents, MailEvents, ExplorerEvents } from '~/types/programs'

/**
 * Главный игровой контроллер урока.
 *
 * Вызывается ОДИН РАЗ в корневом компоненте страницы урока.
 * Внутри использует provide/inject — inject доступны во всех дочерних компонентах.
 *
 * @param config - полный конфиг урока (LessonConfig)
 * @param totalSteps - сколько шагов в уроке (необязательно: дефолт 1)
 */
export function useGameController(config: LessonConfig, totalSteps = 1) {
  // ── Зависимости ──────────────────────────────────────────────────────────
  const userStore = useUserStore()
  const mascotStore = useMascotStore()
  const { triggerGameOver, resetGameOver } = useGameOver()

  // ── Игровое состояние ─────────────────────────────────────────────────────
  const step = ref(0)
  const isCompleted = ref(false)
  const isFailed = ref(false)

  // ── Методы ────────────────────────────────────────────────────────────────

  /**
   * Правильное действие пользователя.
   * Инкрементирует step. Если достигнут последний шаг — вызывает complete().
   *
   * Вызывается из:
   * - QuickAction.onClick (задача 1.22)
   * - ивентов программ (если правильное действие)
   * - явно из кода конфига урока
   */
  function nextStep() {
    if (isFailed.value || isCompleted.value) return

    step.value += 1

    if (step.value >= totalSteps) {
      complete()
    }
  }

  /**
   * Ошибка пользователя.
   *
   * Порядок:
   * 1. Заблокировать контроллер (isFailed = true) → повторные ивенты игнорируются
   * 2. POST /hp/hit (через useUserStore) → обновить hp, isDepleted
   * 3. Сообщить маскоту — эмоция sad, реплика по reason
   * 4. Показать overlay Game Over (useGameOver.triggerGameOver)
   *
   * @param reason - человекочитаемое объяснение: «Вы перешли по фишинговой ссылке»
   */
  async function fail(reason: string) {
    // Идемпотентность: если уже упали — игнорировать повторные ивенты
    if (isFailed.value || isCompleted.value) return

    isFailed.value = true

    // Списать HP — await, чтобы overlay показал актуальное значение
    const hitResult = await userStore.hitHp()

    // Реакция маскота
    mascotStore.setEmotion('sad')
    mascotStore.say(reason, { emotion: 'sad' })

    // Показать Game Over overlay
    // isDepleted — флаг «HP кончились», тогда показываем другой экран
    triggerGameOver({
      reason,
      isDepleted: hitResult?.isDepleted ?? false,
      categoryId: config.categoryId
    })
  }

  /**
   * Урок пройден успешно.
   *
   * Порядок:
   * 1. Заблокировать контроллер (isCompleted = true)
   * 2. POST /lessons/:id/complete (через useUserStore)
   * 3. Маскот — happy
   * 4. Показать экран успеха (navigateTo или локальный overlay — решить в 1.15)
   */
  async function complete() {
    if (isFailed.value || isCompleted.value) return

    isCompleted.value = true

    await userStore.completeLesson(config.id)

    mascotStore.setEmotion('happy')
    mascotStore.say('Отлично! Ты справился с заданием.', {
      emotion: 'happy',
      duration: 4000
    })

    // TODO (1.15): показать CompletionOverlay или navigateTo к списку уроков
    // navigateTo(`/app/${config.categoryId}`)
  }

  /**
   * Сброс состояния (кнопка «Попробовать снова» в GameOver).
   * Возвращает игру к шагу 0, снимает блокировки, убирает overlay.
   */
  function reset() {
    step.value = 0
    isCompleted.value = false
    isFailed.value = false

    mascotStore.setEmotion('neutral')
    mascotStore.clearPhrases()

    resetGameOver()
  }

  // ── Контекст для provide ──────────────────────────────────────────────────

  const ctx: GameControllerContext = {
    step: readonly(step),
    totalSteps,
    isCompleted: readonly(isCompleted),
    isFailed: readonly(isFailed),
    nextStep,
    fail,
    complete,
    reset,
    config
  }

  // ── provide — раздаём контекст вниз ──────────────────────────────────────

  // Основной контекст контроллера (для любого дочернего компонента)
  provide(GAME_CONTROLLER_KEY, ctx)

  // Сырой конфиг (программы могут читать напрямую: данные, чаты и т.д.)
  provide('lessonConfig', config)

  // Ивенты программ — передаём «готовые» обработчики из конфига.
  // Оборачиваем в guard: если контроллер заблокирован — молчим.
  provide('browserEvents', wrapEvents(config.events?.browser))
  provide('messengerEvents', wrapEvents(config.events?.messenger))
  provide('mailEvents', wrapEvents(config.events?.mail))

  // explorerEvents — отдельный тип, но принцип тот же
  provide('explorerEvents', config.events?.explorer ?? null)

  // Данные для программ (чаты, письма, сайты, закладки)
  provide('messengerChats', config.messengerChats ?? [])
  provide('mailEmails', config.mailEmails ?? [])
  provide('mailFolders', config.mailFolders ?? [])
  provide('browserSites', config.browserSites ?? [])
  provide('browserBookmarks', config.browserBookmarks ?? [])

  // QuickActions — для компонента QuickActions (1.22)
  // Оборачиваем onClick каждого в guard isFailed/isCompleted
  provide(
    'quickActions',
    (config.quickActions ?? []).map((action) => ({
      ...action,
      onClick: () => {
        if (!isFailed.value && !isCompleted.value) {
          action.onClick()
        }
      }
    }))
  )

  // ── Инициализация ─────────────────────────────────────────────────────────

  // Синхронизировать HP при старте урока
  onMounted(async () => {
    await userStore.syncHp(config.categoryId)

    // Если HP уже 0 до начала — сразу показать экран depleted
    if (userStore.isDepleted) {
      triggerGameOver({ reason: '', isDepleted: true, categoryId: config.categoryId })
    }
  })

  return ctx
}

// ── Хелпер ───────────────────────────────────────────────────────────────────

/**
 * Оборачиваем объект ивентов: если контроллер заблокирован (isFailed/isCompleted) —
 * ивенты не вызываются. Это защита от «двойного fail» при быстрых кликах.
 *
 * Возвращает новый объект с теми же ключами, но каждый коллбэк обёрнут в guard.
 * Если events = undefined → возвращает null (программа работает без ивентов).
 */
function wrapEvents<T extends Record<string, ((...args: unknown[]) => void) | undefined>>(
  events?: T
): T | null {
  if (!events) return null

  const wrapped = {} as T

  for (const key in events) {
    const fn = events[key]
    if (typeof fn === 'function') {
      // eslint-disable-next-line @typescript-eslint/no-explicit-any
      ;(wrapped as any)[key] = (...args: unknown[]) => {
        fn(...args)
      }
    }
  }

  return wrapped
}
```

## Как программы используют inject

### Browser — пример

```ts
// OS/Programs/Browser/index.vue (или useBrowserTabs.ts)
const events = inject<BrowserEvents | null>('browserEvents', null)
const sites = inject<BrowserSiteConfig[]>('browserSites', [])
const bookmarks = inject<BookmarkConfig[]>('browserBookmarks', [])

// useBrowserSites получает sites при mount:
onMounted(() => {
  if (sites.length) registerSites(sites)
})

// При навигации браузер вызывает:
function navigateTo(url: string) {
  // ... обновить вкладку
  events?.onUrlNavigate?.(url)
}

// При submit формы:
function handleFormSubmit(url: string, data: Record<string, string>) {
  events?.onFormSubmit?.(url, data)
}
```

### Messenger — пример

```ts
// OS/Programs/Messenger/index.vue
const events = inject<MessengerEvents | null>('messengerEvents', null)
const chats = inject<ChatConfig[]>('messengerChats', [])

const messengerState = useMessengerState(events ?? undefined)

// При mount — добавляем чаты урока поверх базовых:
onMounted(() => {
  if (chats.length) messengerState.addChats(chats)
})

// При клике по ссылке в сообщении:
function handleLinkClick(chatId: string, url: string) {
  events?.onLinkClick?.(chatId, url)
}
```

### GameControllerContext в любом компоненте

```ts
import { GAME_CONTROLLER_KEY } from '~/types/game'

// Внутри любого компонента страницы урока:
```

## Гарантии идемпотентности

`fail()` и `complete()` имеют ранний `return` если `isFailed || isCompleted`. Это критично: пользователь может успеть кликнуть по ссылке несколько раз подряд до того как overlay отрендерится. Без guard произошло бы несколько `POST /hp/hit`.

```
Клик → fail() → isFailed = true → POST /hp/hit (1 раз)
             ↘ следующий ивент → fail() → ранний return → игнорируется
```

## Шаги урока (step система)

`step` — целое число, начинается с 0. `totalSteps` — общее количество шагов, передаётся вторым аргументом в `useGameController`. Конкретная семантика шагов — в конфиге урока.

**Пример:** урок с 2 шагами:

```
step 0: пользователь должен проигнорировать ссылку
step 1: пользователь должен использовать URL-checker
```

```ts
// pages/app/office/phishing-intro/index.vue
const { nextStep, fail } = useGameController(phishingIntroConfig, 2)

// phishingIntroConfig.events:
events: {
  browser: {
    onUrlNavigate: (url) => {
      if (url === 'tools://url-checker' && step.value === 1) {
        nextStep() // шаг 1 → complete
      } else if (url.includes('bit.ly')) {
        fail('Вы открыли фишинговый сайт!')
      }
    }
  }
}
```

Если `totalSteps = 1` (большинство уроков) — один `nextStep()` сразу вызывает `complete()`.

## Взаимодействие с useUserStore (1.19)

`useGameController` не трогает HP напрямую. Он вызывает `userStore.hitHp()` и `userStore.completeLesson()`. userStore сам работает с API. Разделение ответственности: контроллер — оркестратор, стор — работает с сетью.

```
fail(reason)
  → await userStore.hitHp()          // POST /hp/hit
  → hitResult.isDepleted             // true если HP = 0
  → triggerGameOver({ isDepleted })  // overlay знает какой экран показать
```

---

## Взаимодействие с useMascotStore (1.20)

Маскот реагирует на ключевые события, но **не блокирует** игровой поток:

```
fail(reason)   → mascotStore.setEmotion('sad')  + mascotStore.say(reason)
complete()     → mascotStore.setEmotion('happy') + mascotStore.say('Отлично!')
nextStep()     → (опционально) mascotStore.say('Молодец, продолжай!', { duration: 2000 })
```

Маскот — cosmetic layer. Если `useMascotStore` в будущем окажется недоступен (inject вернёт null), игра продолжается.

## Взаимодействие с useGameOver (1.14)

`useGameOver` держит реактивный флаг `isVisible` и данные для overlay. Контроллер вызывает `triggerGameOver(payload)` — `useGameOver` сам рендерит overlay через `GameOver/index.vue` (компонент должен быть вставлен в страницу урока рядом с `<OSWorkbench>`).

```ts
// pages/.../[lesson]/index.vue
<template>
  <div class="lesson-page">
    <OSWorkbench />
    <OSTaskbar />
    <OSTaskbarTooltips />
    <GameOver />          ← всегда в DOM, показывается через v-if внутри
    <QuickActions />      ← панель действий (задача 1.22)
    <LessonAlert />       ← подсказка маскота (задача 1.15)
  </div>
</template>
```

`GameOver/index.vue` получает данные через `inject` из `useGameOver` (Pinia store, а не provide — потому что `useGameOver` — singleton, а `GameOver.vue` может быть вне дерева provide).

Кнопка «Попробовать снова» в `GameOver/index.vue`:

```ts
const gameController = inject(GAME_CONTROLLER_KEY)!
function handleRetry() {
  gameController.reset()
}
```
