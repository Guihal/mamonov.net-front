# Задача 1.14: Создание useGameOver

**Статус**: ⬜ Не выполнено

## Описание

Логика проигрыша. При ошибке пользователя (клик по фишинговой ссылке, ввод данных на фейковом сайте, открытие вирусного вложения) — снимается HP и показывается экран Game Over с объяснением, что пошло не так.

Два режима экрана:

1. **Game Over** — HP > 0, ошибка, можно попробовать снова
2. **HP Depleted** — HP = 0, нужно ждать восстановления (таймер обратного отсчёта)

---

## Архитектура

```
useGameController.fail(reason)
  │
  ├── await userStore.hitHp()          ← списать HP
  │        └── обновляет hp, isDepleted в useUserStore
  │
  └── triggerGameOver({ reason, isDepleted, categoryId })
           │
           └── useGameOver
                 ├── isVisible = true      ← показать overlay
                 ├── reason = reason       ← причина провала
                 ├── isDepleted = ...      ← режим экрана
                 └── categoryId = ...     ← для ссылки «К урокам»

GameOver/index.vue
  ├── читает из useGameOver(): isVisible, reason, isDepleted, categoryId
  ├── читает из useUserStore(): hp, maxHp, restoredAt
  │
  ├── [mode: game-over]
  │     ├── Иконка + Заголовок «Ошибка!»
  │     ├── Причина провала (reason)
  │     ├── HP бар (<GameOverHpBar />)
  │     ├── Кнопка «Попробовать снова» → gameController.reset()
  │     └── Кнопка «К урокам» → /app/[categoryId]
  │
  └── [mode: depleted]
        ├── Иконка + Заголовок «HP закончились»
        ├── Текст про восстановление
        ├── <GameOverDepletedTimer /> ← обратный отсчёт
        └── Кнопка «К урокам»
```

## Зависимости

| Что нужно             | Откуда берётся      | Задача |
| --------------------- | ------------------- | ------ |
| `useUserStore`        | `app/stores/`       | 1.19   |
| `GAME_CONTROLLER_KEY` | `app/types/game.ts` | 1.13   |
| `useGameController`   | вызывается выше     | 1.13   |

> `useGameOver` намеренно **не** использует `inject(GAME_CONTROLLER_KEY)` — `reset()` вызывается через коллбэк, который передаётся при `triggerGameOver()`. Это позволяет использовать overlay независимо от контроллера.

## 1. `app/composables/game/useGameOver.ts`

Синглтон через `useState` — чтобы состояние overlay было доступно из любого компонента без prop drilling.

```ts
// app/composables/game/useGameOver.ts

export interface GameOverPayload {
  /** Человекочитаемое объяснение ошибки. Пустая строка при isDepleted. */
  reason: string
  /** true — HP = 0, показываем экран ожидания вместо «Попробовать снова». */
  isDepleted: boolean
  /** categoryId нужен для ссылки «К урокам» → /app/[categoryId]. */
  categoryId: string
  /** Коллбэк «Попробовать снова». Прокидывается из gameController.reset(). */
  onRetry?: () => void
}

// useState — Nuxt composable, реактивен глобально (SSR-safe).
// Используем его вместо ref() на верхнем уровне модуля.
const _isVisible = () => useState<boolean>('gameOver_isVisible', () => false)
const _payload = () => useState<GameOverPayload | null>('gameOver_payload', () => null)

export function useGameOver() {
  const isVisible = _isVisible()
  const payload = _payload()

  /**
   * Показать overlay Game Over.
   * Вызывается из useGameController.fail().
   */
  function triggerGameOver(p: GameOverPayload) {
    payload.value = p
    isVisible.value = true
  }

  /**
   * Скрыть overlay и очистить состояние.
   * Вызывается при нажатии «Попробовать снова» или после navigateTo.
   */
  function resetGameOver() {
    isVisible.value = false
    // Небольшая задержка, чтобы анимация закрытия успела отыграть
    setTimeout(() => {
      payload.value = null
    }, 300)
  }

  return {
    isVisible: readonly(isVisible),
    payload: readonly(payload),
    triggerGameOver,
    resetGameOver
  }
}
```

## 4. `app/components/GameOver/DepletedTimer.vue`

Обратный отсчёт до восстановления HP. Читает `restoredAt` из `useUserStore`.

```vue
<!-- app/components/GameOver/DepletedTimer.vue -->
<template>
  <div class="depleted-timer">
    <template v-if="timeStr">
      <UIcon name="i-lucide-clock" class="depleted-timer__icon" />
      <span class="depleted-timer__label">Восстановление через</span>
      <span class="depleted-timer__value">{{ timeStr }}</span>
    </template>
    <template v-else>
      <span class="depleted-timer__label">HP восстановлены! Перезагрузите страницу.</span>
    </template>
  </div>
</template>

<script setup lang="ts">
const userStore = useUserStore()

// Форматируем оставшееся время в MM:SS
const timeStr = computed(() => {
  const ms = userStore.timeUntilRestore
  if (ms === null || ms <= 0) return null

  const totalSeconds = Math.ceil(ms / 1000)
  const minutes = Math.floor(totalSeconds / 60)
  const seconds = totalSeconds % 60

  return `${String(minutes).padStart(2, '0')}:${String(seconds).padStart(2, '0')}`
})

// Тик каждую секунду пока живёт компонент
let intervalId: ReturnType<typeof setInterval> | null = null

onMounted(() => {
  intervalId = setInterval(() => {
    // Достаточно просто триггернуть реактивность userStore.timeUntilRestore.
    // timeUntilRestore — computed от restoredAt, который уже запоминает дату.
    // Computed пересчитается автоматически, так как вызвана в шаблоне.
    // Но setInterval нужен, чтобы шаблон перерисовывался каждую секунду.
    // Для этого используем дополнительный реактивный тик:
    void userStore.tickTimer()
  }, 1000)
})

onUnmounted(() => {
  if (intervalId !== null) clearInterval(intervalId)
})
</script>

<style lang="scss">
.depleted-timer {
  display: flex;
  align-items: center;
  gap: 6px;
  background: #fef3c7; // amber-100
  border: 1px solid #fde68a; // amber-200
  border-radius: 8px;
  padding: 8px 14px;
}

.depleted-timer__icon {
  width: 16px;
  height: 16px;
  color: #f59e0b; // amber-500
  flex-shrink: 0;
}

.depleted-timer__label {
  font-size: 0.8125rem;
  color: #92400e; // amber-800
}

.depleted-timer__value {
  font-size: 0.9375rem;

  letter-spacing: 0.02em;

```

> **Примечание**: `userStore.tickTimer()` — вспомогательный метод в `useUserStore`, который нужно добавить в задаче 1.19. Он просто инкрементирует внутренний счётчик `_tick`, что форсирует пересчёт `timeUntilRestore`:

```ts
// В useUserStore (задача 1.19) — добавить:
const _tick = ref(0)

const timeUntilRestore = computed(() => {
  void _tick.value // реактивная зависимость
  if (!restoredAtDate.value) return null
  const diff = restoredAtDate.value.getTime() - Date.now()
  return diff > 0 ? diff : null
})

function tickTimer() {
  _tick.value++
}
```

## 6. Поведение кнопок

| Кнопка              | Условие       | Действие                                      |
| ------------------- | ------------- | --------------------------------------------- |
| «Попробовать снова» | `!isDepleted` | `payload.onRetry?.()` → `resetGameOver()`     |
| «К урокам»          | всегда        | `resetGameOver()` → `router.push(/app/catId)` |

При клике «Попробовать снова» `gameController.reset()` выполняет:

```ts
function reset() {
  step.value = 0
  isCompleted.value = false
  isFailed.value = false
  mascotStore.setEmotion('neutral')
  mascotStore.clearPhrases()
  resetGameOver() // закрыть overlay
}
```
